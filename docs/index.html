How the Pipeline Works

10-step automated workflow from paper collection to reproducibility scoring

STEP 01

Collect Papers

Query NASA ADS for astrophysics papers mentioning Jupyter notebooks in the last 2 months.

STEP 02

Classify

Categorise each paper by notebook hosting platform — GitHub, Zenodo, GitLab, personal sites.

STEP 03

Extract Mentions

Fetch arXiv LaTeX tarballs and extract every notebook mention with context, section, and link form.

STEP 04

Clone Repos

Clone GitHub-hosted repositories with shallow depth for efficiency.

STEP 05

Score (RRS)

Static analysis across 5 categories: Environment, Data, Documentation, Portability, Signals.

STEP 06

Detect Python

Detect required Python version from runtime.txt, .python-version, or setup files.

STEP 07

Isolate Environment

Create a per-repo pyenv + venv with all declared dependencies installed.

STEP 08

Execute Notebooks

Run all notebooks via nbconvert, capturing outputs and errors per cell.

STEP 09

Compare Outputs

Diff original vs re-executed notebook outputs at the cell level.

STEP 10

Score (ROS/RCS)

Compute Reproducibility Outcome Score and Composite Score from execution evidence.

Research Questions

Four questions guiding the study of notebook reproducibility in astrophysics

RQ 1

How are notebooks referenced in publications?

Section placement, link form (URL, DOI, footnote), and mention context extracted from full LaTeX source.

RQ 2

How stable are the referenced notebooks?

Execution success rates, error categories, and RRS reproducibility scores across repositories.

RQ 3

Where are notebooks located or stored?

Hosting platform classification — GitHub, Zenodo, GitLab, Binder, personal sites, and others.

RQ 4

How do notebooks receive links and citations?

URL vs DOI vs footnote vs plain-text citation mechanics across astrophysics sub-fields.

Running the Pipeline

Designed for NFDI JupyterHub — no local Docker needed

# 1. Set your NASA ADS token
export ADS_API_TOKEN=your_token_here

# 2. Collect papers from NASA ADS
bash collect.sh

# 3. Extract notebook mentions from arXiv LaTeX source
bash mentions.sh --limit 50

# 4. Clone, score, and execute repos
bash run.sh --count 2

# Or use the Python entry point directly
python3 pipeline/main.py all --count 2 --limit 50
